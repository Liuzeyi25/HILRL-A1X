## 💡 核心技术特性

### 评估流程

```
输入：
  - N 个 checkpoint 路径
  - M 条轨迹 pkl 路径
  
处理：
  1. 加载每个 checkpoint 的 agent 状态
  2. 对每条轨迹：
     a. 提取 obs, actions, rewards, dones
     b. 计算真实折现回报 R_t
     c. 用 agent.forward_critic() 推理 Q_t
     d. 计算误差指标（MAE/RMSE/R）
  3. 聚合统计结果

输出：
  - 图表：q_comparison_traj#.png
  - 图表：q_statistics.png（多模型对比）
  - 目录：trajectory_#_frames/（图像序列）
  - 终端：详细指标表
```

### 指标说明

**MAE** (Mean Absolute Error)：
$$\text{MAE} = \frac{1}{T}\sum_{t=0}^{T-1}|Q_t - R_t|$$
- 范围：≥ 0，越小越好
- 含义：平均估计误差的绝对值

**RMSE** (Root Mean Square Error)：
$$\text{RMSE} = \sqrt{\frac{1}{T}\sum_{t=0}^{T-1}(Q_t - R_t)^2}$$
- 范围：≥ 0，越小越好
- 含义：惩罚大误差，对离群值敏感

**Pearson R** (相关系数)：
$$R = \frac{\text{Cov}(Q, R)}{\sigma_Q \sigma_R}$$
- 范围：[-1, 1]，越接近 1 越好
- 含义：Q 值趋势与真实回报的相关性

## 🎨 可视化详解

### q_comparison_traj#.png（2个子图）

**上子图：Q值曲线对比**
- 黑色实线：实际折现回报（ground truth）
- 彩色线：不同 checkpoint 的 Q 值估计

解读：
- 如果彩色线紧贴黑线 → ✅ Critic 学习得好
- 如果偏离明显 → ⚠️ Critic 需要进一步训练
- 如果完全反向 → ❌ 存在严重问题（如 entropy 崩溃）

**下子图：估计误差（对数尺度）**
- 展示 $|Q_t - R_t|$ 随时间变化
- 对数尺度便于同时观察小误差和大误差
- 下降趋势表示 Critic 在不同部分的学习情况

### q_statistics.png（2个柱状图）

**左图：MAE 对比**
- X 轴：轨迹索引
- Y 轴：Mean Absolute Error
- 柱子颜色：不同 checkpoint

用途：快速比较不同 checkpoint 的平均表现

**右图：RMSE 对比**
- 类似 MAE，但惩罚大误差
- 用于识别容易出现大误差的轨迹或 checkpoint

### trajectory_#_frames/（图像序列）

轨迹的采样图像帧，用途：
- 验证轨迹数据的可视化
- 制作演示视频（使用 ffmpeg）
- 理解任务执行过程

可用命令制作视频：
```bash
# 将帧转换为视频
ffmpeg -framerate 10 -i trajectory_0_frames/frame_%03d.png output.mp4
```

| `--save_video_samples` | flag | ✗ | False | 保存图像帧 |
| `--ensemble_agg` | str | ✗ | `min` | Q 聚合：min / mean |
| `--chunk_size` | int | ✗ | 16 | 批推理大小 |
| `--gamma` | float | ✗ | 0.99 | 折现因子 |
| `--use_target_critic` | flag | ✗ | False | 使用 target critic |
| `--n_frames_per_traj` | int | ✗ | 50 | 每轨迹帧数 |

### 1. 验证训练进度
```bash
python3 scripts/evaluate_q_values.py \
    --model_paths ckpt_1k ckpt_5k ckpt_10k ckpt_20k \
    --trajectory_paths eval_traj.pkl \
    --output_dir ./progress_analysis
```
**观察**：Q 值 MAE 应逐步降低 ✓

### 2. 诊断训练问题
```bash
python3 scripts/evaluate_q_values.py \
    --model_paths problematic_ckpt \
    --trajectory_paths eval_traj.pkl \
    --output_dir ./diagnosis
```
**诊断**：
- MAE 不变或升高 → 学习率问题？
- Q 值完全反向 → Entropy 崩溃？
- 高方差 → 数据分布变化？

### 3. 对比算法变体
```bash
python3 scripts/evaluate_q_values.py \
    --model_paths bc_v1_ckpt bc_v2_ckpt orpo_ckpt \
    --trajectory_paths test_traj.pkl \
    --output_dir ./algorithm_comparison
```
**对比**：哪个算法的 Critic 学习最好
